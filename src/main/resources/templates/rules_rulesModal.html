Dividend Eligibility Processing Rules

Calculation: eligibleQuantity = settledQuantity - onLoanQuantity

If eligibleQuantity < 0, set to 0 and log a warning.

This rule determines how many shares are actually eligible to receive the real dividend, excluding shares that are currently on loan.

If pendingSettlementQuantity > 0: Cum-dividend trade (buy) settling after record date

  • claimQuantity = pendingSettlementQuantity
  • Dividend claim must be raised against selling counterparty

If pendingSettlementQuantity < 0: Pending sell trade

  • Shares remain with lender at record date
  • No adjustment to eligible quantity

If pendingSettlementQuantity = 0: No action required

Look up withholding tax rule for client type and domicile.

If client is taxExempt AND treaty rate is 0:

  • Apply 0% withholding tax
  • Client receives full exemption

If treatyRatePercent < statutoryRatePercent:

  • Apply treaty rate (reduced from statutory)
  • Client benefits from tax treaty between countries

Otherwise:

  • Apply statutory rate
  • No treaty relief available

Calculations:

  • grossRealDividend = eligibleQuantity × dividendPerShare
  • withholdingTaxAmount = grossRealDividend × (withholdingTaxRateApplied ÷ 100)
  • netRealDividend = grossRealDividend - withholdingTaxAmount

If onLoanQuantity > 0: Process securities lending position

  • grossManufacturedDividend = onLoanQuantity × dividendPerShare
  • Dividend received as manufactured payment, not real dividend

Manufactured Dividend Tax Treatment:

  • Always apply statutory rate (treaty relief does NOT apply)
  • manufacturedWithholdingTaxAmount = grossManufacturedDividend × (statutoryRate ÷ 100)
  • netManufacturedDividend = grossManufacturedDividend - manufacturedWithholdingTaxAmount

Gross-Up Processing:

If grossUpAgreed = true:

  • grossUpAmount = manufactured WHT charged - treaty WHT that would have applied
  • Borrower obligated to pay additional amount to compensate for lost treaty benefit
  • netManufacturedDividendAfterGrossUp = netManufacturedDividend + grossUpAmount

If grossUpAgreed = false:

  • grossUpAmount = 0
  • Tax leakage borne by lender
  • netManufacturedDividendAfterGrossUp = netManufacturedDividend

Calculation: dividendClaimAmount = claimQuantity × dividendPerShare

If claimQuantity > 0, this represents the gross dividend amount to be raised as a claim against the counterparty for cum-dividend trades.

Claims are typically raised through the custodian's claims process to recover dividend entitlement from trades that settle after the record date.

Calculation:

totalNetEntitlement = netRealDividend + netManufacturedDividendAfterGrossUp + dividendClaimAmount

This represents the total amount the client will ultimately receive after all tax deductions, gross-ups, and claims are processed.

Status Determination Logic:

  • MANUFACTURED_ONLY: eligibleQuantity = 0 AND onLoanQuantity > 0 AND claimQuantity = 0
  • INELIGIBLE: eligibleQuantity = 0 AND onLoanQuantity = 0
  • CLAIM_REQUIRED: claimQuantity > 0 AND eligibleQuantity > 0
  • PARTIALLY_ELIGIBLE: onLoanQuantity > 0 AND eligibleQuantity > 0
  • FULLY_ELIGIBLE: None of the above conditions met

The status provides a quick visual indicator of the client's overall eligibility position and any required actions.
